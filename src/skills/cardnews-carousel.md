## Card News Carousel — Direction Guide

구현 정확성보다 먼저 **"고밀도 에디토리얼 카드뉴스"**를 우선한다.
PPT처럼 보이면 실패. 카드뉴스 디자이너가 만든 것처럼 보여야 한다.

---

## 1. 디자인 철학 (MOST IMPORTANT)

### 시각 덩어리감 (Visual Chunking)
- 텍스트, 카드, 색면이 **2~3개의 큰 덩어리**로 묶여야 한다
- 작은 컴포넌트를 안전하게 나열하는 방식 금지
- 제목+서브가 하나의 덩어리, 항목들이 하나의 큰 카드 덩어리, 강조가 하나의 바 덩어리
- **덩어리 사이에만 간격, 덩어리 안은 촘촘하게**

### 타이포가 주인공
- 제목은 정보 전달 수단이 아니라 **화면의 중심 시각 요소**
- 한국어 문장이 길면 폰트를 줄이지 말고, **과감하게 줄바꿈**
- 제목 블록이 화면의 30~40%를 먹어야 한다
- "경쟁사가 베끼는 건" + 줄바꿈 + "기술이 아닙니다" — 이런 식으로 덩어리 유지

### 위계 차이 (Hierarchy Contrast)
- 요소 수가 부족한 게 아니라, **큰 것과 작은 것의 차이**가 약한 게 문제
- 제목(72~92px)과 본문(30px)의 크기 차이를 극대화
- 강조 박스는 얇은 바가 아니라 **큰 면**으로
- 번호 배지(48px)와 라벨(18px)의 대비

### 구획 컬러 (Zoning Color)
- 색이 장식처럼 붙는 게 아니라, **"이 부분은 하나의 카드", "이 부분은 강조 섹션"** 구분
- 컬러 바가 얇게 붙고 카드가 작으면, 색이 있어도 비어 보인다
- 컬러 바/강조 박스를 "붙이는 요소"가 아니라 **화면 구획 면**으로 키울 것

### 편집 리듬 (Editorial Rhythm)
- 헤더, 카드, 색상 바, 강조 박스, pill — 이것들이 같은 간격 체계 안에서 움직여야
- 모든 요소가 비슷한 크기면 리듬이 없다 → 큰/중/작 3단계 크기 위계

## 2. 씬별 레이아웃

### Cover (표지)
- 구성: 에피소드 pill → 대제목 덩어리 → 구분선 → 서브카피 박스 → 배경 워드마크 (5덩어리)
- 좌측 정렬
- **대제목이 화면의 35~40%를 차지** — 72~92px, weight 800~900
- 줄바꿈은 의미 단위로 과감하게 (한 줄 10~12자)
- 배경에 초대형 반투명 워드마크 (200~400px, opacity 0.04)
- 하단 서브카피는 **독립 패널**(bg card, 패딩 넉넉히)로 강조
- 빈 공간 → 큰 배경 패널이나 기하학 장식으로 메움
- **상단에 정보 몰리고 하단이 비는 구도 금지** — 세로로 고르게 분배

### List (세로 목록)
- 구성: 타이틀 → 설명 카피 → 큰 카드 덩어리(항목들) → 강조 면 (4덩어리)
- **타이틀 아래 설명 1~2줄** 추가 (키워드는 accent 색)
- **4개 항목이 하나의 큰 카드 덩어리** 안에 들어감
- 카드 덩어리 = bg #0a1628, rounded-xl, 화면 너비의 88%, **높이는 화면의 45~50%**
- 각 항목 높이 110~120px — 카드가 콘텐츠에 맞춰 줄어들지 말 것
- 좌측 16px 컬러 바 + emoji(32px) + 라벨(30~38px) + 우측 pill
- 항목 사이 약한 구분선(1px, #ffffff0d)
- 카드 덩어리 **바로 아래에** accent 배경 강조 면(높이 120~160px) — 띄우지 말고 붙여서 묶음

### Split (2열 비교)
- 구성: 타이틀 → 설명 1~2줄 → 좌우 카드 → 중앙 VS 장치 → 하단 결론 면 (5덩어리)
- **타이틀 아래에 반드시 설명 카피 2~3줄** 추가 (건조하지 않게)
- 좌우 2개 **큰 카드**가 각각 화면 너비의 42~44%
- 카드 높이: **화면의 32~38%** — 1350px 기준 최소 430px. 절대 줄이지 말 것
- 카드 내부: 항목을 배지/아이콘/짧은 설명 포함 한 줄 블록으로, 촘촘하게
- **중앙에 VS 배지 또는 세로 구분선** 필수 (비교 긴장감)
- 좌: 붉은 톤 border + 경고 느낌 + ✕ 아이콘
- 우: accent 톤 border + 해결 느낌 + ✓ 아이콘
- 하단 accent 결론 면: **카드 바로 아래에 붙여서** 하나의 묶음으로. 바닥까지 내리지 말 것

### Flow (프로세스)
- 3개 step이 **수직 연결 구조**
- 번호 원형 배지(48px, accent bg) + 연결선(2px, accent) 필수
- 각 step 카드는 화면 너비의 80%+, 높이 120px+
- 단순 가로 나열 금지 → 리듬감 있는 배치

### Focus (단일 강조)
- 구성: 배경 패널 → 큰 따옴표 장식 → quote 블록 → accent side bar → 출처 (5덩어리)
- quote를 **매우 크게** (96~120px, weight 800)
- 큰 따옴표 장식 (160~200px, accent, opacity 0.15) — 시선 고정 장치
- accent side bar (좌측 4px, 높이 = quote 높이)
- 반투명 배경 패널 (bg #0a1628, 화면의 70% 차지)
- quote만 덩그러니 두지 말고, **장식+패널로 밀도감 확보**
- 화면 중앙에 quote 패널이 **주인공**으로 자리잡아야 함

### Closing (마무리)
- "BRAND" 초대형 배경 워드마크 (300px+, opacity 0.04)
- CTA 박스: accent 배경, 큰 면으로, 전면 하단 배치
- URL pill: accent 배경, 28px
- 면책 2줄: 14px, 50% opacity
- **강한 엔딩 카드** 느낌

## 3. 타이포그래피

| 역할 | 사이즈 | Weight |
|------|--------|--------|
| Cover 타이틀 | 72~92px | 800~900 |
| Body 타이틀 | 52~64px | 800 |
| Body 텍스트/항목 | 30~38px | 400~600 |
| Quote | 96~120px | 800 |
| 태그/pill | 22~26px | 500 |
| 캡션 | 18~22px | 300 |

**한국어 규칙:**
- 폰트를 줄여서 해결하지 말고 **줄바꿈과 블록 크기로 해결**
- 한 줄 최대 10~12자 (한글 기준, 타이틀)
- 의미 단위로 과감한 줄바꿈 → 시각적 덩어리 유지
- 한글은 반드시 const 변수에 넣고 JSX에서 참조

## 4. 색상

### 태정 스타일 (기본)
```
배경:        #1a1a2e
카드 배경:   #0a1628
장식 패널:   #2d2d44
악센트:      #00AEEF
텍스트:      #ffffff
텍스트 서브:  #ffffffb3
보더:        #ffffff1a
```

### 항목별 구획 컬러 (list 모드)
```
1: #ef4444 → 배경 #ef444426
2: #f97316 → 배경 #f9731626
3: #8b5cf6 → 배경 #8b5cf626
4: #ec4899 → 배경 #ec489926
5: #22c55e → 배경 #22c55e26
```

**8-digit hex만 사용. rgba 금지.**

## 5. 장식 레이어

모든 씬에 **최소 2개** 사용:
- 배경 카드 패널 (#0a1628) — 콘텐츠를 감싸는 큰 면
- 오버사이즈 워드마크 (200~400px, opacity 0.04)
- 미세 그리드 라인 (1px accent, 60px 간격, opacity 0.06)
- accent side bar (좌측 3~4px)
- 숫자 배지 (원형 accent, 40~48px)
- 큰 따옴표 기호 (160~200px)
- gradient overlay (섹션 구분용)

## 6. 구현 규칙

- `<Series>`로 슬라이드 연결. 각 durationInFrames: 90
- 별도 함수 컴포넌트. `export const MyAnimation = () => { ... };`
- useCurrentFrame(). spring + interpolate + clamp만 사용
- 8-digit hex만. rgba 금지. React.FC 금지
- 폰트: `Inter, system-ui, sans-serif`

### 애니메이션
- snappy spring: { damping: 15, stiffness: 200, durationInFrames: 25 }
- 시퀀싱: 배경(0f) → 타이틀(5f) → 서브(10f) → 항목(15f, 간격 4f) → 강조(50f)

---

## 7. 금지 사항

- ❌ 작은 컴포넌트를 안전하게 나열하는 방식
- ❌ 텍스트만 배경 위에 띄우기 (패널/카드 위에 배치)
- ❌ 카드 높이를 콘텐츠에 맞춰 줄이기 (최소 높이 강제)
- ❌ 타이틀 폰트를 줄여서 해결
- ❌ 모든 씬을 동일 레이아웃으로 반복
- ❌ 과도한 중앙 정렬
- ❌ 요소들을 균등분포로 띄엄띄엄 배치
- ❌ 컬러 바를 얇은 장식으로만 사용 (구획 면으로 키울 것)
- ❌ 캔버스 80% 미만 시각적 사용
- ❌ 상단에 정보 몰리고 하단 50%가 비는 구도
- ❌ 강조 박스를 본문과 떨어뜨려 푸터처럼 배치
- ❌ 비교 카드에 VS/구분선 없이 그냥 나란히 두기
- ❌ "채우지 못해서 남은 여백" — 여백은 반드시 "강조를 위한 여백"이어야 함
